                                                  # Frontmatter

                                                  Any Markdown file that contains a YAML frontmatter block will be processed by gray-matteropen in new window. The frontmatter must be at the top of the Markdown file, and must take the form of valid YAML set between triple-dashed lines. Example:

                                                  ---
title: Docs with VitePress
editLink: true
---

                                                  Between the triple-dashed lines, you can set predefined variables, or even create custom ones of your own. These variables can be used via the special $frontmatter variable.

                                                  Hereâ€™s an example of how you could use it in your Markdown file:

                                                  ---
title: Docs with VitePress
editLink: true
---

# {{ $frontmatter.title }}

Guide content

                                                  # Alternative Frontmatter Formats

                                                  VitePress also supports JSON frontmatter syntax, starting and ending in curly braces:

                                                  ---
{
  "title": "Blogging Like a Hacker",
  "editLink": true
}
---
